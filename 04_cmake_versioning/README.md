
# Versioning - CMake


Here is the example of versioning in cmake

## Create a config file to store version

```bash
configure_file(<input> <output> [COPYONLY] [ESCAPE_QUOTES] [@ONLY] [NEWLINE_STYLE List])
```
The configure_file options are 
1. `HelloConfig.h.in` : This is the input file that will be processed by the `configure_file()` command
2. `HelloConfig.h` : This is the output file that will be generated by the `configure_file()` command in build 

There are more params to the `configure_file()` command check the documentation for more information : [configure_file](https://cmake.org/cmake/help/latest/command/configure_file.html)

The version is defined in `CMakeLists.txt` via `project(... VERSION x.y.z)`. CMake's `configure_file()` then substitutes the version variables (e.g. `@HelloCmake_VERSION_MAJOR@`) in the `.h.in` template to generate a header file (`HelloConfig.h`) that your C++ code can include.
